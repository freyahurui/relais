# Relais 产品需求文档 (PRD)

---

## 1. 产品概述

### 1.1 产品定位

Relais 是一款 AI 增强的智能信息中转站工具，旨在通过智能化技术消除信息整理的负担，为用户提供极致的收集、识别和归档体验。

### 1.2 核心价值主张

**智能归档** — AI 自动判断内容类型（灵感/笔记/待办/日程），无需手动选择文件夹即可完成归档

**简约高效** — 粘贴即存，AI 自动识别时间、地点等关键信息

### 1.3 产品名称

**正式名称**：Relais（智能中转站）

**名称由来**：Relais 源自法语，意为"驿站"或"中继站"，寓意临时存放、各取所需的信息中转体验。

### 1.4 一句话描述

> 一个智能中转站：随手粘贴，AI 自动分类存放，消息、日程、待办各取所需，告别备忘录和聊天框的杂乱存储。

## 2. 目标用户

### 2.1 典型使用场景

| 场景 | 用户行为 | Relais 价值 |
|------|----------|-------------|
| **通知转日程** | 收到会议/活动通知，复制消息粘贴到 App | AI 自动识别时间、地点，智能生成日程事项 |
| **随手记灵感** | 突然有灵感想法，快速记录 | 简单输入后自动归档到「灵感」分类 |
| **待办管理** | 记录需要完成的任务 | AI 识别截止时间，自动归类为待办 |
| **通知消息归档** | 收到重要通知消息，需要保存备查 | 粘贴即存，智能提取关键信息并归档 |

### 3.2 内容收集模块

| 输入方式 | 说明 | 状态 |

**输入体验优化**（已实现）：
- 用户只需输入内容，无需手动选择分类
- 提交后自动触发 AI 分析，智能识别内容类型和分类
- AI 自动判断是日程、待办还是笔记
- 根据 AI 识别结果自动归档到对应分类

### 3.3 AI 智能处理模块

| 能力 | 说明 | 状态 |
|------|------|------|
| 时间/日期识别 | 识别「明天下午3点」「2月15日」等时间表达 | ✅ 已实现 |
| 地点识别 | 识别「会议室A」「XX咖啡厅」等地点信息 | ✅ 已实现 |
| 内容类型判断 | 判断内容属于日程、待办还是笔记 | ✅ 已实现 |
| 自动分类 | 根据内容性质自动归类到对应分类 | ✅ 已实现 |
| 关键信息提取 | 提取核心信息（活动名称、参与人等） | ✅ 已实现 |

**AI 分类规则**（已实现）：

| 内容类型 | 自动归档分类 | 示例 |
|----------|-------------|------|
| Todo（待办事项） | 💼 工作 | "完成项目报告，本周五截止" |
| Schedule（日程安排） | 💼 工作 | "明天下午3点开会，地点在会议室A" |
| Inspiration（灵感记录） | 💡 灵感 | "突然想到一个好点子" |
| 学习相关 | 📚 学习 | "《三体》读后感" |
| 生活相关 | 🏠 生活 | "周末去超市购物清单" |

**AI 技术方案**：
- **提供商**：OpenAI 兼容 API（通过 openai-next.com 中转）
- **模型**：gpt-5-mini
- **调用方式**：通过 Supabase Edge Functions 代理调用
- **响应格式**：结构化 JSON（type、category、时间、地点、置信度）

**归档入口**：

| 入口 | 说明 |
|------|------|

| 详情页 📦 按钮 | 编辑详情页头部的归档按钮 |
| 设置页开关 | 自动归档总开关，状态持久化到 localStorage |

## 4. 用户体验设计

### 4.1 设计原则

- **简约而不简单**：界面简洁，但交互细节精致
- **内容优先**：减少视觉噪音，让用户聚焦于内容本身
- **流畅自然**：动效和过渡符合用户直觉

### 4.2 配色方案

**整体风格**：莫兰迪色系，温柔优雅，低调高级

| 颜色类型 | 色值参考 | 应用场景 |
|----------|----------|----------|
| 主背景色 | `#FAF7F2` / `#F5F1E8` | 页面背景（米白色） |
| 次级背景 | `#F0EBE0` | 卡片、分组背景（浅灰褐） |
| 主文字色 | `#4A4540` | 标题、正文（深褐灰） |
| 次要文字 | `#9B958C` | 辅助信息、时间戳（中灰褐） |
| 强调色 | `#9CB4B8` | 按钮、链接、选中状态（莫兰迪蓝） |

### 4.4 核心交互流程

```
1. 打开 App
   ↓
2. 点击输入框或浮动 + 按钮
   ↓
3. 弹出输入模态框
   ↓
4. 用户粘贴/输入内容
   ↓
5. 点击「完成」提交
   ↓
6. AI 后台处理中（异步，不阻塞）
   ↓
7. 项目立即保存到默认分类
   ↓
8. AI 处理完成后自动更新类型、分类、时间等字段
```

## 5. 技术需求

### 5.1 技术架构

```
┌─────────────────────────────────────────────────┐
│                   用户界面层                      │
│          (React 18 + Tailwind CSS)               │
├─────────────────────────────────────────────────┤
│                   业务逻辑层                      │
│  ┌──────────┬──────────┬──────────┬──────────┐  │
│  │内容收集  │AI 处理   │分类归档  │日程管理  │  │
│  └──────────┴──────────┴──────────┴──────────┘  │
├─────────────────────────────────────────────────┤
│                   数据持久层                      │
│           (Supabase - PostgreSQL)                │
├─────────────────────────────────────────────────┤
│  ┌──────────────────┬─────────────────────────┐  │
│  │   AI 能力集成    │      认证授权           │  │
│  │  Edge Functions  │    Supabase Auth        │  │
│  │   + OpenAI API   │                         │  │
│  └──────────────────┴─────────────────────────┘  │
└─────────────────────────────────────────────────┘
```

### 5.2 技术栈

| 类别 | 技术选型 | 说明 |
|------|----------|------|
| 开发语言 | JavaScript (ES6+) | React 18 Hooks |
| UI 框架 | React 18 + Tailwind CSS | 组件化开发 |
| 状态管理 | React Hooks | useState, useEffect, useCallback |
| 数据库 | Supabase (PostgreSQL) | 云端数据库 + Auth |
| AI 服务 | OpenAI 兼容 API (gpt-5-mini) | 通过 Edge Functions 调用 |
| 认证 | Supabase Auth | JWT token 认证 |

### 5.3 数据存储方案

**数据库表结构**：

```sql
-- 分类表
CREATE TABLE categories (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  name TEXT NOT NULL,
  emoji TEXT NOT NULL,
  sort_order INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 项目表
CREATE TABLE items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  category_id UUID REFERENCES categories(id),
  title TEXT NOT NULL,
  content TEXT,
  item_type TEXT CHECK (item_type IN ('todo', 'schedule', 'note')),
  scheduled_start TIMESTAMPTZ,
  scheduled_end TIMESTAMPTZ,
  location TEXT,
  is_completed BOOLEAN DEFAULT FALSE,
  completed_at TIMESTAMPTZ,
  ai_parsed_data JSONB,
  confidence_score DECIMAL(3,2),
  source_type TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  deleted_at TIMESTAMPTZ,
  archived_at TIMESTAMPTZ DEFAULT NULL,
  is_auto_archived BOOLEAN DEFAULT FALSE
);

-- 活动日志表
CREATE TABLE activity_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  action TEXT NOT NULL,
  item_type TEXT,
  item_id UUID,
  details JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```
