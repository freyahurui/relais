# Relais 产品需求文档 (PRD)

---

## 1. 产品概述

### 1.1 产品定位

Relais 是一款与 iOS 系统备忘录和日历共存的 AI 增强工具，旨在通过智能化技术消除信息整理的负担，为用户提供极致的收集、识别和归档体验。

### 1.2 产品角色

Relais 承担三个核心角色：

| 角色 | 描述 |
|------|------|
| **智能日程管理助手** | 自动识别通知消息中的时间、地点信息，智能创建日程提醒 |
| **智能备忘录** | 统一收集灵感、笔记、通知等各类文本内容，自动分类归档 |
| **智能文件传输助手** | 临时存放各类文件，支持图片识别和智能整理 |

### 1.3 核心价值主张

**通知转日程** — 针对通知消息场景，AI 自动识别时间、地点等关键信息，智能生成日程事项

**智能归档** — AI 自动判断内容类型（灵感/笔记/待办/文件），无需手动选择文件夹即可完成归档

**简约私密** — 纯本地存储保障数据隐私，遵循 Apple 设计语言，提供简约高级的使用体验

### 1.4 产品名称

**正式名称**：Relais（智能中转站）

**名称由来**：Relais 源自法语，意为"驿站"或"中继站"，寓意临时存放、各取所需的信息中转体验。

### 1.5 一句话描述

> 一个智能中转站：随手粘贴，AI 自动分类存放，消息、日程、待办、文件各取所需，告别备忘录和聊天框的杂乱存储。

## 2. 目标用户

### 2.1 用户群体

Relais 面向广泛的日常用户群体，尤其是那些需要频繁处理信息、管理日程，但希望减少手动整理成本的 iOS 用户。

### 2.2 典型使用场景

| 场景 | 用户行为 | Relais 价值 |
|------|----------|----------------|
| **通知转日程** | 收到会议/活动通知，复制消息粘贴到 App | AI 自动识别时间、地点，智能生成日程事项 |
| **随手记灵感** | 突然有灵感想法，快速记录 | 简单输入后自动归档到「灵感」分类 |
| **临时存放文件** | 需要保存临时文件或图片，稍后整理 | 统一存放，AI 根据内容类型自动分类 |
| **通知消息归档** | 收到重要通知消息，需要保存备查 | 粘贴即存，智能提取关键信息并归档 |

### 2.3 用户痛点

- **信息散落各处**：灵感、通知、待办事项散落在备忘录、微信、日历等多个应用中
- **手动整理繁琐**：需要手动提取通知中的时间地点信息，手动选择文件夹分类
- **隐私顾虑**：担心敏感信息上传到云端服务

### 2.4 用户需求特征

- 追求效率，希望减少重复操作
- 重视数据隐私和安全感
- 偏爱简约、美观的产品设计

**AI 分类规则**（已实现）：

| 内容类型 | 自动归档分类 | 示例 |
|----------|-------------|------|

**AI 技术方案**：
- **提供商**：智谱 AI（Zhipu GLM）
- **模型**：GLM-4-flash（免费额度，响应快速）
- **调用方式**：通过 Supabase Edge Functions 代理调用
- **响应格式**：结构化 JSON（type、category、时间、地点、置信度）

## 4. 用户体验设计

### 4.1 设计原则

- **遵循 Apple 设计语言**：采用 Human Interface Guidelines 的设计规范
- **简约而不简单**：界面简洁，但交互细节精致
- **内容优先**：减少视觉噪音，让用户聚焦于内容本身
- **流畅自然**：动效和过渡符合用户直觉

### 4.2 配色方案

**整体风格**：莫兰迪色系，温柔优雅，低调高级

| 颜色类型 | 色值参考 | 应用场景 |
|----------|----------|----------|
| 主背景色 | `#FAF7F2` / `#F5F1E8` | 页面背景（米白色） |
| 次级背景 | `#F0EBE0` | 卡片、分组背景（浅灰褐） |
| 主文字色 | `#4A4540` | 标题、正文（深褐灰） |
| 次要文字 | `#9B958C` | 辅助信息、时间戳（中灰褐） |
| 强调色 | `#9CB4B8` | 按钮、链接、选中状态（莫兰迪蓝） |
| 成功色 | `#B8C4A8` | 成功状态（莫兰迪绿） |
| 警告色 | `#D4C4A8` | 警告状态（莫兰迪黄） |
| 错误色 | `#C8A8A8` | 错误状态（莫兰迪红） |
| 分组色 | `#C8B8A8` / `#B8C4C8` / `#D4C8D4` | 不同分类的柔和标识 |

**配色设计要点**：
- 采用低饱和度的莫兰迪色系，营造温柔优雅的视觉感受
- 所有颜色带有灰调，避免鲜艳刺眼
- 大量留白 + 暖调背景，营造温暖舒适的氛围
- 精细的灰度层级区分信息重要性
- 强调色克制使用，仅用于关键操作
- 渐变和阴影轻柔自然，与莫兰迪色系呼应
- 整体色调和谐统一，减少视觉疲劳

### 4.5 核心交互流程

**收集与归档流程（当前简化版）**：

```
1. 打开 App
   ↓
2. 点击浮动 + 按钮或输入框
   ↓
3. 弹出输入模态框
   ↓
4. 用户粘贴/输入内容
   ↓
5. 点击「完成」提交
   ↓
6. AI 后台处理中（异步，不阻塞）
   ↓
7. 项目立即保存到默认分类
   ↓
8. AI 处理完成后自动更新：
   - 修改 item_type（todo/schedule/inspiration）
   - 更新 category_id（自动分类）
   - 填充时间、地点等字段
   ↓
9. 用户可手动点击项目查看/编辑
```

**完整编辑流程**：

```
1. 点击任意项目卡片
   ↓
2. 全屏编辑界面弹出
   ↓
3. 根据项目类型显示对应字段：
   - Todo: 截止日期、完成状态
   - Schedule: 开始/结束时间、地点
   - Inspiration: 仅标题和内容
   ↓
4. 可修改分类选择器
   ↓
5. 点击右上角 ✓ 保存
   ↓
6. 返回列表，显示更新后内容
```

### 5.3 AI 能力集成

**实现方式**：调用云端 AI API

| 能力 | 实现方案 |
|------|----------|
| 内容分析 | 调用 LLM API 进行内容理解和分类 |

**API 调用流程**：
```
用户输入内容
    ↓
构造 Prompt（包含系统提示词 + 用户内容）
    ↓
调用 AI API
    ↓
解析结构化响应（JSON）
    ↓
展示预览界面
```

**隐私保护**：
- API 请求不包含用户身份信息
- AI 服务端不保存用户数据
- 仅传输必要的分析内容

**数据库表结构**：

```sql
-- 分类表
CREATE TABLE categories (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  name TEXT NOT NULL,
  emoji TEXT NOT NULL,
  sort_order INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 项目表
CREATE TABLE items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  category_id UUID REFERENCES categories(id),
  title TEXT NOT NULL,
  content TEXT,
  item_type TEXT CHECK (item_type IN ('todo', 'schedule', 'inspiration')),
  scheduled_start TIMESTAMPTZ,
  scheduled_end TIMESTAMPTZ,
  location TEXT,
  is_completed BOOLEAN DEFAULT FALSE,
  completed_at TIMESTAMPTZ,
  ai_parsed_data JSONB,
  confidence_score DECIMAL(3,2),
  source_type TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  deleted_at TIMESTAMPTZ
);

-- 附件表
CREATE TABLE attachments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  item_id UUID REFERENCES items(id) ON DELETE CASCADE,
  file_name TEXT NOT NULL,
  file_type TEXT,
  file_url TEXT,
  file_size INTEGER,
  storage_path TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 活动日志表
CREATE TABLE activity_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  action TEXT NOT NULL,
  item_type TEXT,
  item_id UUID,
  details JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```
